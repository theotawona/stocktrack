# StockTrack Learner Handout (1 Page)

## Purpose
Use this handout during daily work to complete tasks in the correct order.

## Workflow Order (Always Follow This)
1. Request stock
2. Manager reviews and approves
3. Issue approved stock
4. Report usage and attach evidence

## Role Quick Guide

### Staff
- Go to My Requisitions
- Enter clear purpose and urgency
- Add stocked items from dropdown
- Add unlisted items for procurement needs
- Submit requisition
- Track status and review notes
- Add usage report after work is completed

### Manager
- Go to Requisition Approvals
- Review request purpose and lines
- Approve quantities realistically
- Add review note (especially when reducing quantity)
- Go to Issue Stock and issue against approved requisitions only
- Check shortfalls and trigger restock action

### Admin
- Maintain properties, storerooms, suppliers, and stock items
- Manage users, roles, and property assignments
- Monitor issuance logs, reorder list, and reconciliation history

## Status Meaning
- Pending: waiting for manager review
- Approved: approved and ready for issue
- Partially Issued: some approved quantity issued
- Issued: all approved quantity issued
- Rejected: not approved
- Cancelled: requester cancelled

## Common Mistakes to Avoid
- Submitting requisition with no clear purpose
- Issuing stock before approval
- Ignoring shortfall/restock-needed results
- Using vague notes like "needed urgently" with no detail

## Daily Quality Checklist
- Purpose is specific and actionable
- Quantities are realistic
- Unlisted items include useful notes/specifications
- Comments are professional and clear
- Usage report is submitted after stock is used

## If You See Errors

### Could not submit requisition
- Confirm purpose is entered
- Confirm at least one item is added (stocked or unlisted)

### Cannot issue requested quantity
- Issue available quantity only
- Use restock-needed output to plan procurement

### Missing menu options
- Ask admin to check your role and property assignment
- Log out and log in again

## Support Escalation Path
1. Team lead / supervisor
2. Property manager
3. System admin
