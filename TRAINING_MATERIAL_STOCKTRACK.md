# StockTrack Training Material

## 1. Training Overview

### Audience
- Staff users
- Managers
- Admin users

### Duration
- Full session: 2 hours 15 minutes
- Staff-only version: 60 minutes
- Manager-only version: 75 minutes
- Admin-only version: 90 minutes

### Training Goal
By the end of training, every user can complete their daily workflow in StockTrack without support.

### Learning Outcomes
Participants will be able to:
- Log in and move confidently through the menu
- Find stock, interpret status, and use filters
- Create and track requisitions
- Approve and issue requisitions correctly (manager/admin)
- Manage stock master data and users safely (admin)
- Report usage and upload supporting documents

---

## 2. Session Flow (Easy and Practical)

### Module 1: Welcome and System Basics (15 min)
- Explain what StockTrack is used for
- Explain user roles and who does what
- Show menu layout and where to find key pages
- Explain important statuses:
  - Pending
  - Approved
  - Partially Issued
  - Issued
  - Rejected
  - Cancelled

### Module 2: Staff Workflow (35 min)
- Stock page: view available items and low stock
- My Requisitions page:
  - Create a requisition
  - Add stocked items
  - Add unlisted items (procurement needed)
  - Submit requisition
  - Track status updates
- Add usage reports, comments, and supporting documents

### Module 3: Manager Workflow (35 min)
- Requisition Approvals page:
  - Review pending requests
  - Distinguish stocked lines from procurement lines
  - Approve quantities and add review notes
- Issue Stock page:
  - Issue only against approved requisitions
  - Understand partial issuing when stock is not enough
  - Read shortfall and restock-needed outputs

### Module 4: Admin Workflow (30 min)
- Properties and Storerooms setup
- Suppliers and Stock master data maintenance
- Users page:
  - Create users
  - Assign roles
  - Assign staff to a property
- Reconciliation basics and good control habits

### Module 5: Assessment and Q&A (20 min)
- Short practical challenge per role
- Recap common mistakes and best practices
- Open questions

## 3. Role-Based Daily Workflows

## Staff Daily Flow
1. Log in
2. Open Stock and check item availability
3. Open My Requisitions
4. Create requisition details (purpose and urgency)
5. Add stocked items
6. Add unlisted items if needed
7. Submit requisition
8. Monitor status and manager feedback
9. When stock is received and used, add usage report
10. Upload evidence documents if required

## Manager Daily Flow
1. Log in
2. Open Requisition Approvals
3. Review each pending requisition
4. Approve, partially approve, or reject with clear note
5. Open Issue Stock
6. Select approved requisition and issue available quantities
7. Review shortfall and restock-needed table
8. Coordinate procurement for missing items

## Admin Daily Flow
1. Log in
2. Review overview and critical alerts
3. Maintain master data (properties, storerooms, suppliers, stock)
4. Maintain users and role assignments
5. Audit issuance and requisition activity
6. Run reconciliation checks

## 4. Trainer Script (What to Say)

### Opening Script (3 minutes)
Welcome everyone. Today we are learning StockTrack by following the same steps you do in real work. We will keep this practical. By the end of this session, you should be able to complete your role-specific tasks confidently.

### Transition to Staff Module
We start with the staff journey because this is where demand begins: identifying what is needed and submitting clear requisitions.

### Transition to Manager Module
Now that requests are submitted, we move to manager actions: review, approve, and issue with stock controls.

### Transition to Admin Module
Finally, we cover admin controls that keep data clean and secure: users, setup, and governance.

### Closing Script (2 minutes)
Great work. The key to success is following the workflow order: request, approve, issue, then report usage. If every role does this consistently, StockTrack stays accurate and reliable.

## 5. Hands-On Exercises

## Exercise A: Staff Requisition with Mixed Basket
Scenario:
A cleaner needs 10 toilet rolls and 2 mops, but mops are not currently in stock.

Tasks:
1. Create a new requisition
2. Add toilet rolls as stocked item
3. Add mops as unlisted item with note: heavy duty handle
4. Submit requisition
5. Confirm the requisition appears in My Requisitions

Expected result:
- Requisition is created successfully
- One stocked line and one procurement line are visible

## Exercise B: Manager Approval and Issue
Scenario:
Review the requisition from Exercise A and issue what is available.

Tasks:
1. Open Requisition Approvals
2. Approve stocked line quantity
3. Keep procurement line noted for sourcing
4. Open Issue Stock and issue approved quantity
5. Check shortfall/restock-needed output

Expected result:
- Requisition status updates correctly
- Issuance does not reduce stock below zero

## Exercise C: Admin User Assignment
Scenario:
A new staff member joins Rosebank View.

Tasks:
1. Create new user
2. Assign role as staff
3. Assign property as Rosebank View
4. Save and verify user appears in list

Expected result:
- User can only access allowed staff pages
- User sees only their assigned property data

## 6. Common Errors and Fixes

## Error: Could not submit requisition
Likely cause:
- Missing purpose
- No items in basket
- Invalid or empty item details

Fix:
- Confirm purpose is filled
- Ensure at least one stocked or unlisted item is added
- Re-submit

## Error: Cannot issue requested quantity
Likely cause:
- Stock available is lower than approved quantity

Fix:
- Issue available quantity only
- Use restock-needed list to trigger procurement

## Error: User cannot see expected menu
Likely cause:
- Role permissions or property assignment mismatch

Fix:
- Admin checks user role and property assignment in Users page
- User logs out and logs in again

## 7. Quick Reference for Learners

### Status Meaning
- Pending: waiting for review
- Approved: approved and ready for issue
- Partially Issued: some quantity has been issued
- Issued: all approved quantity issued
- Rejected: request not approved
- Cancelled: request withdrawn by requester

### Golden Rules
- Always enter a clear purpose
- Use unlisted items for procurement needs
- Never bypass approval before issuing
- Treat comments and notes as official communication
- Keep data clean and specific
